Table List

  • Addresses
        The Addresses table contains fields for holding complete address. Used with infrastructures, pollution source site and contacts from municipalities.
  • AppErrLogs
        The AppErrLogs table contains fields for holding application errors.
  • AppTasks
        The AppTasks table contains fields for holding application tasks sent to the server which can take a long time to complete.
  • AppTaskLanguages
        The AppTaskLanguages table contains fields for holding bilingual texts for AppTasks table.
  • BoxModels
        The BoxModels table contains fields for holding and managing Box Model scenarios
  • BoxModelLanguages
        The BoxModelLanguages table contains fields for holding bilingual texts for BoxModels table.
  • BoxModelResults
        The BoxModelResults contains fields for holding box modelling scenarios results.
  • Classifications
        The Classifications table contains fields for holding classification for visualization on the web. It holds only a representation of the actual legal classification held in Arc GIS.
  • ClimateDataValues
        The ClimateDataValues table contains fields for holding climate data values for various climate site from Meteorological Service of Canada.
  • ClimateSites
        The ClimateSites table contains fields for holding climate site related information from Meteorological Service of Canada as well as some provincial climate sites.
  • CoCoRaHSSites
        The CoCoRaHSSites table contains fields for holding voluntary managed precipitation sites from private individuals.
  • CoCoRaHSValues
        The CoCoRaHSValues table contains fields for holding precipitation values from voluntary private sites.
  • Contacts
        The Contacts table contains fields for holding users information.
  • ContactPreferences
        The ContactPreferences table contains fields for maintaining users preferences while using some applications.
  • ContactShortcuts
        The ContactShortcuts table contains fields for maintaining users shortcuts used by some applications.
  • DocTemplates
        The DocTemplates table contains fields for holding preset templates while creating many type of documents, like .xlsx, .docx, .csv, .txt, etc. This option of exporting data to documents is being replaced by parameter type exports. This table could be removed in the future.
  • DrogueRuns
        The DrogueRuns table contains fields for showing drogue tracking on the web. It holds the names, numbers and date of the drogue runs.
  • DrogueRunPositions
        The DrogueRunPositions table contains fields for showing drogue tracking on the web. It contains the temporal coordinates.
  • Emails
        The Emails table contains fields for holding complete emails. It is used with municipality contacts and second emails from users.
  • EmailDistributionLists
        The EmailDistributionLists table contains fields for identifying email distribution lists.
  • EmailDistributionListContacts
        The EmailDistributionListContacts table contains all contacts names for a specified email distribution list.
  • EmailDistributionListContactLanguages
        The EmailDistributionListContactLanguages table contains fields for holding bilingual texts from EmailDistributionListContacts table.
  • EmailDistributionListLanguages
        The EmailDistributionListLanguages table contains fields for holding bilingual texts from EmailDistributionLists.
  • HelpDocs
        The HelpDocs table contains fields for holding help information used in some applications. Especially the web.
  • HydrometricDataValues
        The HydrometricDataValues table contains fields for holding hydrometric data values for various hydrometric site from Hydrometric Service of Canada.
  • HydrometricSites
        The HydrometricSites table contains fields for holding hydrometric site related information from Hydrometric Service of Canada as well as some provincial hydrometric sites.
  • Infrastructures
        The Infrastructures table contains fields for holding all related information for municipal infrastructures. Both WWTP and LS related information are stored in the same table.
  • InfrastructureLanguages
        The InfrastructureLanguages table contains fields for holding bilingual texts for the infrastructure table.
  • LabSheets
        The LabSheets table contains fields for storing and managing lab sheets.
  • LabSheetDetails
        The LabSheetDetails table contains fields for holding lab sheet detail related information.
  • LabSheetTubeMPNDetails
        The LabSheetTubeMPNDetails table contains fields for holding lab tube information.
  • Logs
        The Logs table is used to hold a history of the users commands to the database.
  • MapInfos
        The MapInfos table contains fields for holding mapping related information for all objects in the database.
  • MapInfoPoints
        The MapInfoPoints table contains fields for holding mapping coordinates related information.
  • MikeBoundaryConditions
        The MikeBoundaryConditions table contains fields for holding MIKE boundary condition related information.
  • MikeScenarios
        The MikeScenarios table contains fields for holding and managing MIKE scenarios.
  • MikeScenarioResults
        The MikeScenarioResults table contains fields for holding a portion of the MIKE scenario results. Limited use at this time. Don’t know if this table will be kept in the future. Currently all MIKE scenario results are stored in files.
  • MikeSources
        The MikeSources table contains fields for holding and managing MIKE sources which could be rivers, streams, brooks or other pollution sources.
  • MikeSourceStartEnds
        The MikeSourceStartEnds table contains fields for non continuous spills.
  • MWQMAnalysisReportParameters
        The MWQMAnalysisReportParameters table contains fields for managing analysis report parameters.
  • MWQMLookupMPNs
        The MWQMLookupMPNs table contains all possible MPNs.
  • MWQMRuns
        The MWQMRuns table contains fields for holding and managing a marine water quality monitoring runs.
  • MWQMRunLanguages
        The MWQMRunLanguages table contains fields for holding bilingual texts for the MWQMRuns table.
  • MWQMSamples
        The MWQMSamples table contains fields for holding sample results for a marine water quality site and run.
  • MWQMSampleLanguages
        The MWQMSampleLanguages table contains fields for holding bilingual texts for the MWQMSamples table.
  • MWQMSites
        The MWQMSites table contains fields for holding and managing marine water quality monitoring sites.
  • MWQMSiteStartEndDates
        The MWQMSiteStartEndDates table contains fields for storing the marine water quality monitoring site activity dates.
  • MWQMSubsectors
        The MWQMSubsectors table contains fields for describing the marine water quality monitoring subsectors.
  • MWQMSubsectorLanguages
        The MWQMSubsectorLanguages table contains fields for holding bilingual texts for the MWQMSubsectors table.
  • PolSourceGroupings
        The PolSourceGroupings table contains fields for pollution source grouping which is used to formulate sentence from selected options while describing pollution source sites and issues.
  • PolSourceGroupingLanguages
        The PolSourceGroupingLanguages table contains fields for holding bilingual texts for the PolSourceGroupings table.
  • PolSourceObservations
        The PolSourceObservations table contains fields for holding the pollution source observation related information.
  • PolSourceObservationIssues
        The PolSourceObservationIssues table contains fields for holding the description of the pollution source observation issues.
  • PolSourceSites
        The PolSourceSites table contains fields for holding and managing pollution source sites.
  • PolSourceSiteEffects
        The PolSourceSiteEffects table contains fields for establishing the effects of pollution source site.
  • PolSourceSiteEffectTerms
        The PolSourceSiteEffectTerms table contains all the terms used to describe the effects of pollution source site.
  • RainExceedances
        The RainExceedances table contains fields for managing rain exceedances.
  • RainExceedanceClimateSites
        The RainExceedanceClimateSites table contains fields for holding the links between rain exceedance and climate sites.
  • RatingCurves
        The RatingCurves table contains fields for storing rating curves for each hydrometric sites.
  • RatingCurveValues
        The RatingCurveValues table contains fields for holding rating curve values for each hydrometric sites.
  • ReportSections
        The ReportSections table contains fields for holding report section used to produce various reports.
  • ReportTypes
        The ReportTypes table contains fields for holding the report types that can be produce with just a click of a button.
  • ResetPasswords
        The ResetPasswords table contains basic fields to help with the reset of users password.
  • SamplingPlans
        The SamplingPlans table contains fields for organizing sampling plans. It’s also the primary table use to create sampling plan text files that can be distributed to contractors, provinces and internal users. They use it with the application CSSPWQInputTool.
  • SamplingPlanEmails
        The SamplingPlanEmails table contains fields for holding emails needed to be sent while filling lab sheets.
  • SamplingPlanSubsectors
        The SamplingPlanSubsectors table contains fields for storing sampling plan subsector information.
  • SamplingPlanSubsectorSites
        The SamplingPlanSubsectorSites table contains fields for storing and managing sampling plan subsector marine water quality monitoring sites.
  • Spills
        The Spills table contains fields for holding information regarding spills from municipal infrastructure or other types of spills.
  • SpillLanguages
        The SpillLanguages table contains fields for holding bilingual texts for the Spills table.
  • Tels
        The Tels table contains fields for holding telephone numbers. It is used with municipal contacts and users extra profile information.
  • TideDataValues
        The TideDataValues table contains fields for holding values of tide data at various places in Canada and US.
  • TideLocations
        The TideLocations table contains fields for storing the tide locations and names to be used by some applications.
  • TideSites
        The TideSites table contains fields for holding and managing tide site related information from Canadian Hydrographic Services.
  • TVFiles
        The TVFiles table contains fields for managing files outside of the databases via links and file names.
  • TVFileLanguages
        The TVFileLanguages table contains fields for holding bilingual texts for the TVFiles table.
  • TVItems
        The TVItems table contains fields for holding and managing the tree view system within the whole database so each item can be search and is unique.
  • TVItemLanguages
        The TVItemLanguages table contains fields for holding bilingual texts for the TVItems table.
  • TVItemLinks
        The TVItemLinks table contains fields for linking different tree view items (TVItems) between them.
  • TVItemStats
        The TVItemStats table contains fields for holding statistics of various tree view items within the database. It is use by some applications as an indicator of how many sub items exist underneath a particular item.
  • TVItemUserAuthorizations
        The TVItemUserAuthorizations table contains fields for managing the authorization of item a user can view, edit, create or delete.
  • TVTypeUserAuthorizations
        The TVTypeUserAuthorizations table contains fields for managing the authorization of item types a user can view, edit, create or delete.
  • UseOfSites
        The UseOfSites table contains fields for identifying climate sites, hydrometric sites, tide sites, municipalities and drogues to be used under a subsector.
  • VPAmbients
        The VPAmbients table contains fields for storing ambient inputs for Visual Plumes scenarios.
  • VPResults
        The VPResults table contains fields for holding Visual Plumes scenario results.
  • VPScenarios
        The VPScenarios table contains fields for holding and managing Visual Plumes scenarios.
  • VPScenarioLanguages
        The VPScenarioLanguages table contains fields for holding bilingual texts for the VPScenarios table.
