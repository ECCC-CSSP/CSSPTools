Liste des tables

  • Addresses
        La table Addresses contient des champs pour contenir l'adresse complète. Elle est utilisée avec les infrastructures, les sites sources de pollution et les contacts des municipalités.
  • AppErrLogs
        La table AppErrLogs contient des champs pour contenir les erreurs d'application.
  • AppTasks
        La table AppTasks contient des champs pour contenir les tâches d'application envoyées au serveur qui peuvent prendre un certain temps.
  • AppTaskLanguages
        La table AppTaskLanguages contient des champs pour contenir des textes bilingues pour la table AppTasks.
  • BoxModels
        Le table BoxModels contient des champs pour contenir les scénarios de modélisation 'Box Model'.
  • BoxModelLanguages
        La table BoxModelLanguages contient des champs pour contenir des textes bilingues pour la table BoxModels.
  • BoxModelResults
        Le BoxModelResults contient des champs pour contenir les résultats des scénarios de modélisation de boîtes.
  • Classifications
        La table Classifications contient des champs permettant de conserver la classification pour la visualisation sur le Web. Il ne contient qu'une représentation de la classification juridique réelle détenue dans Arc GIS.
  • ClimateDataValues
        La table ClimateDataValues contient des champs permettant de conserver les valeurs des données climatiques pour divers sites climatiques du Service météorologique du Canada.
  • ClimateSites
        La table ClimateSites contient des champs permettant de conserver les informations relatives aux sites climatiques du Service météorologique du Canada ainsi que de certains sites climatiques provinciaux.
  • CoCoRaHSSites
        La table CoCoRaHSSites contient des champs permettant de conserver les sites de précipitations gérés volontairement par des particuliers.
  • CoCoRaHSValues
        La table CoCoRaHSValues contient des champs permettant de conserver les valeurs de précipitations provenant de sites privés volontaires.
  • Contacts
        La table Contacts contient des champs pour stocker les informations des utilisateurs.
  • ContactPreferences
        La table ContactPreferences contient des champs permettant de gérer les préférences des utilisateurs lors de l'utilisation de certaines applications.
  • ContactShortcuts
        La table ContactShortcuts contient des champs permettant de gérer les raccourcis utilisateurs utilisés par certaines applications.
  • DocTemplates
        La table DocTemplates contient des champs pour contenir des modèles prédéfinis lors de la création de nombreux types de documents, tels que .xlsx, .docx, .csv, .txt, etc. Cette option d'exportation de données vers des documents est remplacée par des exportations de type paramètre. Cette table pourrait être supprimé à l'avenir.
  • DrogueRuns
        La table DrogueRuns contient des champs pour afficher le suivi des drogues sur le Web. Il contient les noms, les numéros et la date des courses de drogue.
  • DrogueRunPositions
        La table DrogueRunPositions contient des champs pour afficher le suivi des drogues sur le Web. Il contient les coordonnées temporelles.
  • Emails
        La table Emails contient des champs pour contenir des e-mails complets. Il est utilisé avec les contacts de la municipalité et les seconds e-mails des utilisateurs.
  • EmailDistributionLists
        La table EmailDistributionLists contient des champs permettant d'identifier les listes de distribution d'e-mails.
  • EmailDistributionListContacts
        La table EmailDistributionListContacts contient tous les noms de contacts pour une liste de distribution d'e-mails spécifiée.
  • EmailDistributionListContactLanguages
        La table EmailDistributionListContactLanguages contient des champs pour contenir les textes bilingues de la table EmailDistributionListContacts.
  • EmailDistributionListLanguages
        La table EmailDistributionListLanguages contient des champs permettant de stocker des textes bilingues d'EmailDistributionLists.
  • HelpDocs
        La table HelpDocs contient des champs pour contenir les informations d'aide utilisées dans certaines applications. Surtout le Web.
  • HydrometricDataValues
        La table HydrometricDataValues contient des champs permettant de conserver les valeurs de données hydrométriques pour divers sites hydrométriques du Service hydrométrique du Canada.
  • HydrometricSites
        La table HydrometricSites contient des champs pour contenir des informations sur les sites hydrométriques du Service hydrométrique du Canada ainsi que certains sites hydrométriques provinciaux.
  • Infrastructures
        La table Infrastructures contient des champs pour contenir toutes les informations relatives aux infrastructures municipales. Les informations relatives à la STEP et au LS sont stockées dans la même table.
  • InfrastructureLanguages
        La table InfrastructureLanguages contient des champs permettant de stocker des textes bilingues pour la table d'infrastructure.
  • LabSheets
        La table LabSheets contient des champs pour stocker et gérer les feuilles de laboratoire.
  • LabSheetDetails
        La table LabSheetDetails contient des champs pour contenir les informations relatives aux détails de la feuille de laboratoire.
  • LabSheetTubeMPNDetails
        La table LabSheetTubeMPNDetails contient des champs pour contenir les informations sur les tubes de laboratoire.
  • Logs
        La table Logs est utilisée pour conserver un historique des commandes des utilisateurs vers la base de données.
  • MapInfos
        La table MapInfos contient des champs pour contenir les informations relatives au mappage pour tous les objets de la base de données.
  • MapInfoPoints
        La table MapInfoPoints contient des champs pour contenir les informations relatives aux coordonnées de mappage.
  • MikeBoundaryConditions
        La table MikeBoundaryConditions contient des champs pour contenir les informations relatives aux conditions aux limites MIKE.
  • MikeScenarios
        La table MikeScenarios contient des champs permettant de conserver et de gérer les scénarios MIKE.
  • MikeScenarioResults
        La table MikeScenarioResults contient des champs pour contenir une partie des résultats du scénario MIKE. Utilisation limitée pour le moment. Je ne sais pas si ce tableau sera conservé à l’avenir. Actuellement, tous les résultats des scénarios MIKE sont stockés dans des fichiers.
  • MikeSources
        La table MikeSources contient des champs pour conserver et gérer les sources MIKE qui peuvent être des rivières, des ruisseaux, des ruisseaux ou d'autres sources de pollution.
  • MikeSourceStartEnds
        La table MikeSourceStartEnds contient des champs pour les déversements non continus.
  • MWQMAnalysisReportParameters
        La table MWQMAnalysisReportParameters contient des champs permettant de gérer les paramètres du rapport d'analyse.
  • MWQMLookupMPNs
        La table MWQMLookupMPNs contient tous les MPN possibles
  • MWQMRuns
        La table MWQMRuns contient des champs permettant de conserver et de gérer une analyse de la qualité de l'eau de mer.
  • MWQMRunLanguages
        The MWQMRunLanguages table contains fields for holding bilingual texts for the MWQMRuns table.
  • MWQMSamples
        La table MWQMSamples contient des champs permettant de conserver les résultats des échantillons pour un site et une analyse de qualité de l'eau de mer.
  • MWQMSampleLanguages
        La table MWQMSampleLanguages contient des champs permettant de stocker des textes bilingues pour la table MWQMSamples.
  • MWQMSites
        La table MWQMSites contient des champs permettant de conserver et de gérer les sites de surveillance de la qualité de l'eau de mer.
  • MWQMSiteStartEndDates
        La table MWQMSiteStartEndDates contient des champs pour stocker les dates d'activité du site de surveillance de la qualité de l'eau de mer.
  • MWQMSubsectors
        La table MWQMSubsectors contient des champs pour décrire les sous-secteurs de surveillance de la qualité de l'eau de mer.
  • MWQMSubsectorLanguages
        La table MWQMSubsectorLanguages contient des champs permettant de stocker des textes bilingues pour la table MWQMSubsectors.
  • PolSourceGroupings
        La table PolSourceGroupings contient des champs pour le regroupement des sources de pollution qui est utilisé pour formuler une phrase à partir des options sélectionnées tout en décrivant les sites et les problèmes de source de pollution.
  • PolSourceGroupingLanguages
        La table PolSourceGroupingLanguages contient des champs pour contenir des textes bilingues pour la table PolSourceGroupings.
  • PolSourceObservations
        La table PolSourceObservations contient des champs pour contenir les informations relatives à l'observation de la source de pollution.
  • PolSourceObservationIssues
        La table PolSourceObservationIssues contient des champs pour contenir la description des problèmes d'observation des sources de pollution.
  • PolSourceSites
        La table PolSourceSites contient des champs permettant de conserver et de gérer les sites sources de pollution.
  • PolSourceSiteEffects
        La table PolSourceSiteEffects contient des champs permettant d'établir les effets du site source de pollution.
  • PolSourceSiteEffectTerms
        La table PolSourceSiteEffectTerms contient tous les termes utilisés pour décrire les effets du site source de pollution.
  • RainExceedances
        La table RainExceedances contient des champs permettant de gérer les dépassements de pluie.
  • RainExceedanceClimateSites
        La table RainExceedanceClimateSites contient des champs pour contenir les liens entre le dépassement de la pluie et les sites climatiques.
  • RatingCurves
        La table RatingCurves contient des champs pour stocker les courbes d'évaluation pour chaque site hydrométrique.
  • RatingCurveValues
        La table RatingCurveValues contient des champs permettant de conserver les valeurs des courbes de notation pour chaque site hydrométrique.
  • ReportSections
        La table ReportSections contient des champs pour contenir la section de rapport utilisée pour produire divers rapports.
  • ReportTypes
        La table ReportTypes contient des champs pour contenir les types de rapport qui peuvent être produits avec un simple clic sur un bouton.
  • ResetPasswords
        La table ResetPasswords contient des champs de base pour aider à la réinitialisation du mot de passe des utilisateurs.
  • SamplingPlans
        La table SamplingPlans contient des champs permettant d'organiser les plans d'échantillonnage. C'est également la table principale utilisée pour créer des fichiers texte de plan d'échantillonnage qui peuvent être distribués aux sous-traitants, aux provinces et aux utilisateurs internes. Ils l'utilisent avec l'application CSSPWQInputTool.
  • SamplingPlanEmails
        La table SamplingPlanEmails contient des champs pour contenir les e-mails à envoyer lors du remplissage des feuilles de laboratoire.
  • SamplingPlanSubsectors
        La table SamplingPlanSubsectors contient des champs pour stocker les informations du sous-secteur du plan d'échantillonnage.
  • SamplingPlanSubsectorSites
        La table SamplingPlanSubsectorSites contient des champs pour stocker et gérer les sites de surveillance de la qualité des eaux marines du sous-secteur du plan d'échantillonnage.
  • Spills
        Le tableau Spills contient des champs pour contenir des informations concernant les déversements provenant des infrastructures municipales ou d'autres types de déversements.
  • SpillLanguages
        La table SpillLanguages contient des champs pour contenir des textes bilingues pour la table Spills.
  • Tels
        La table Tels contient des champs pour les numéros de téléphone. Il est utilisé avec les contacts municipaux et les informations de profil supplémentaires des utilisateurs.
  • TideDataValues
        La table TideDataValues contient des champs pour conserver les valeurs des données de marée à divers endroits au Canada et aux États-Unis.
  • TideLocations
        La table TideLocations contient des champs pour stocker les emplacements de marée et les noms à utiliser par certaines applications.
  • TideSites
        La table TideSites contient des champs permettant de conserver et de gérer les informations relatives au site de marée provenant des Services hydrographiques du Canada.
  • TVFiles
        La table TVFiles contient des champs permettant de gérer des fichiers en dehors des bases de données via des liens et des noms de fichiers.
  • TVFileLanguages
        La table TVFileLanguages contient des champs permettant de stocker des textes bilingues pour la table TVFiles.
  • TVItems
        La table TVItems contient des champs permettant de conserver et de gérer le système d'arborescence dans la base de données de conservation afin que chaque élément puisse être recherché et soit unique.
  • TVItemLanguages
        La table TVItemLanguages contient des champs pour contenir des textes bilingues pour la table TVItems.
  • TVItemLinks
        La table TVItemLinks contient des champs permettant de lier différents éléments de l'arborescence (TVItems) entre eux.
  • TVItemStats
        La table TVItemStats contient des champs permettant de conserver les statistiques de divers éléments de l'arborescence dans la base de données. Il est utilisé par certaines applications comme indicateur du nombre de sous-éléments sous un élément particulier.
  • TVItemUserAuthorizations
        La table TVItemUserAuthorizations contient des champs permettant de gérer l'autorisation d'un élément qu'un utilisateur peut afficher, modifier, créer ou supprimer.
  • TVTypeUserAuthorizations
        La table TVTypeUserAuthorizations contient des champs permettant de gérer l'autorisation des types d'élément qu'un utilisateur peut afficher, modifier, créer ou supprimer.
  • UseOfSites
        La table UseOfSites contient des champs permettant d'identifier les sites climatiques, les sites hydrométriques, les sites de marées, les municipalités et les drogues à utiliser dans un sous-secteur.
  • VPAmbients
        La table VPAmbients contient des champs pour stocker les entrées ambiantes pour les scénarios Visual Plumes.
  • VPResults
        La table VPResults contient des champs permettant de stocker les résultats du scénario Visual Plumes.
  • VPScenarios
        La table VPScenarios contient des champs permettant de conserver et de gérer les scénarios Visual Plumes.
  • VPScenarioLanguages
        La table VPScenarioLanguages contient des champs pour contenir des textes bilingues pour la table VPScenarios.
